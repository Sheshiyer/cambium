# Cambium R3F Visual Moodboard Prompt

Create a polished visual moodboard image for Cambium, a future React Three Fiber / Three.js scene direction. This is NOT a code spec and NOT a literal implementation diagram. It is a visual art-direction board that should help later scene design.

Use the attached dark cartographic UI references for mood only: dense black-green dashboard surfaces, pale chartreuse labels, topographic rings, perspective grids, telemetry chips, condensed display typography, tiny monospaced data, and calm signal/strata language. Do not reproduce the word TERRAIN or any exact layout from the references.

Use the local Codrops/Taste references for direction only: shader-driven graph objects, flat-to-spatial object fields, topographic map framing, dithered particle maps, and scroll-depth/gallery composition. The result should feel renderable in R3F: mostly planes, extruded glyphs, instanced abstract objects, arcs, rails, ribbons, rings, particles, and labels.

Brand essence:
- Cambium is the thin living growth layer where businesses grow.
- "An idea goes in. A self-running, on-brand business comes out."
- It is a company compiler: Genesis -> Taste -> Build -> Ops, with Cortex memory feeding every stage.
- The feel is venture-operator, bioluminescent calm, fluid motion, organic-precise type.
- The scene should feel like a living system of organs working together as modular islands.

Core visual concept:
- A two-mode visual board.
- Left/main panel: "2.5D OVERVIEW" -- a shallow isometric plane or tilted grid with several modular islands connected by hyphae-like signal rails. Each island is an abstract Coolshapes-inspired object, not a building: star, polygon, moon, ellipse, flower, wheel, triangle, capsule, folded rectangle. They should look like simple future R3F node meshes with subtle grainy material and slight extrusion.
- Right/detail panel: "FLAT NODE VIEW" -- a flatter top-down plane zoomed into one selected island, showing the current user position in the process, a breathing ring halo, nearby packets, and compact telemetry.
- Lower strip or side column: a material/shape language board showing "GENESIS", "TASTE", "BUILD", "OPS", and "CORTEX" as labeled glyph specimens.

Palette and constraints:
- Use only this Cambium palette plus opacity: #00272B, #012F34, #E0FF4F, #D6FFF6, #231651, off-cream, and a small peach/warning tint.
- No purple/blue AI gradients. No neon outer glow. No city, houses, towers, streets, robots, spaceships, or literal terrain mountains.
- Lighting should be inner/tinted, calm, and bioluminescent; use inset glints, small halos, and softened material edges.
- Preserve readability: large heading "CAMBIUM"; exact labels only: "CAMBIUM", "2.5D OVERVIEW", "FLAT NODE VIEW", "CORTEX", "GENESIS", "TASTE", "BUILD", "OPS", "ARC X", "9/17".
- Typography should be condensed sans for headings and tiny mono for telemetry. Avoid serif except no more than one small italic annotation if needed.

Composition:
- 1536x1024 landscape moodboard.
- Dark charcoal/teal background with faint grid and topographic contour text-path feeling.
- Use asymmetric editorial framing: large left overview scene, smaller right zoom scene, bottom specimen strip.
- Include enough negative space that it feels premium, not a crowded cockpit.
- The final image should read as a high-end visual direction board for an R3F scene, with a clear future interaction model but without code, UI wireframe boxes, or excessive explanatory text.

Quality bar:
- Sophisticated, specific, and brand-native.
- Abstract objects should be beautiful and renderable, not photoreal.
- The process map should show "where the user is" through one highlighted island, ring orbit, path progress, and compact status chips.
- It should feel like Cambium, not generic SaaS, generic sci-fi, or a copied Terrain screen.
