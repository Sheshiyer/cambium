🧬 genesis — mint the brand

The brand brain. One idea goes in; a full brand system comes out across waves 1–8. Genesis is the first contraction of the organism — it establishes x*, the brand-DNA fixed point that every downstream organ is pulled toward, and writes the seed taste into the cortex.

Sheshiyer/brandmint-oracle-aleph genesis / brand brain idea → brand-dna 💲 gated

1Overview

idea → full brand system. Genesis takes a single seed idea and a thin brand-config.yaml and expands it into a complete brand system — positioning, voice, visual identity, named scenarios, and rendered assets — across an orchestrated run of waves 1–8. Text skills fire first and hydrate the config; visual waves then render against it. The output of this expansion is the setpoint x*: the brand-DNA the rest of the body converges to. Nothing downstream can be more coherent than what genesis mints here, which is why this is the organism's first and defining contraction.

2Internal architecture — the self-similar Φ

The shape repeats at every scale: one input is gathered into a conductor (wave_planner), fanned out to specialist spokes, reconciled, and emitted as a single coherent artifact. A spec-kit seeds the blueprint up front; the cortex persists the resulting taste as x*.

3Components (real, grounded)

One card per real core module — names exactly as they appear in brandmint/core/ (and scripts/ for asset_registry).

wave_planner — the conductor; computes the wave 1–8 plan from config, scenario & depth.
context_analyzer — reads the idea/config and derives the launch context that steers planning.
scenario_recommender — picks the named scenario (e.g. crowdfunding-lean) the brand is built for.
compositor — composites layered visual assets into finished brand imagery.
asset_registry — domain-aware asset selection engine with optional registry layering.
asset_provenance — tracks where each asset came from so outputs stay auditable.
hydrator — maps text-skill outputs back into brand-config.yaml fields for visual prompts.
agent_scaffolder — builds brand-aware agent scaffolds from the hydrated brand config.
skills_registry — the catalogue of text/visual skills the waves can invoke.
kickstarter_blueprint — the spec-kit: a Kickstarter prototype blueprint seeding the run.
coverage_preflight — scenario/depth coverage preflight for publish-quality runs.
asset_mode — resolves the asset generation mode (placeholder vs. paid render).
cache — caches expensive intermediate results across waves.
image_utils — low-level image helpers (load, resize, encode) for the visual waves.
design_memory — the cortex client; persists/reads brand taste (writes x*).
8 subpackages. The package is organised as automation · cli · core · installer · models · pipeline · publishing · vision. The 15 modules above live in core/ (with asset_registry in scripts/); the CLI surface lives in cli/ and the rendering/QA logic in vision/.

5Homeostasis role

Genesis establishes x*. In the homeostatic model, x* is the brand-DNA fixed point that every other organ — hands, taste, will, cortex — is contracted toward. Genesis is where x* first comes into existence: the wave 1–8 expansion both defines the setpoint and seeds the cortex (design_memory.py) with the brand's taste so later runs can converge instead of re-deriving. A non-contractive genesis — one that drops the mission and lets the seed drift — is the classic failure mode: the "Fitcheck reads as fitness" miss, where a fitness-brand idea collapsed into a generic try-on/"fit check" because the contraction toward the real mission wasn't enforced at the seed. Get x* wrong here and no downstream organ can recover it.
