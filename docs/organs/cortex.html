🧠 shared cortex — the aesthetic memory

the 1024-dim metric space the whole engine converges in — where on-brand-ness stops being a vibe and becomes a measurable distance.

taste-nim + DESIGN_MEMORY_WORKER cortex / the metric space feeds all stages ⟳ cross-cutting · gated

1Overview

The cortex is a 1024-dim NIM embedding space. Text is projected by the model nv-embedqa-e5-v5; images are projected by NV-CLIP into the same space, so a render and its brief are comparable. In this space cosine distance is on-brand-ness — d(a, b) = 1 − cos(a, b), and "on-brand" simply means small d. Every organ reads from and writes to it: genesis lays down the setpoint x*, taste scores against it, build pulls the nearest references, and ops feeds outcomes back. It is the one substrate the entire pipeline shares.

2Internal architecture — the self-similar Φ

One Φ shape, fractally reused: the same embed → measure d → pull nearest loop runs inside every organ, every cluster, every venture — and because the metric space is shared, a lesson learned at one scale is readable at all of them.

3Components (real, grounded)

nv-embedqa-e5-v5 — the 1024-dim text embed model (NVIDIA NIM). Indexes with input_type: passage, searches with query; caps at ~512 tokens.
NV-CLIP — image embeddings (nvidia/nvclip), sharing the space with the text model for cross-modal retrieval (a render vs. its brief).
embed — x → vector. Any artifact or brand becomes a point in (X, d).
search — kNN nearest on-brand references; returns the closest points by cosine distance.
taste index — the Codrops crawl; the corpus of "good design" the engine measures against.
design-memory — the brand's seed taste = x*, the setpoint every stage converges toward.
nim.mjs client — skill-clusters: embedText / embedImage / vlmAnnotate against integrate.api.nvidia.com.
design_memory.py client — brandmint: posts /search to the Design Memory Worker, returns local reference image paths (fail-open to []).
embed_worker.py client — snow-gloves: chunks ingest plans and computes NVIDIA embeddings; dimension: 1024 read from snowgloves.yaml.
nim-probe / embed client — skill-clusters scripts: connectivity probe and CLI embed entrypoints over the NIM lane.

4Data flow + contract

Any artifact or brand → embed → a 1024-dim vector → d(vec, x*) = drift; search returns the nearest on-brand references. The contract is directional across the pipeline:

stagerelationship to the cortex
genesiswrites x* — lays down the brand-DNA setpoint.
tastereads & scores — computes d(x, x*) as the on-brand score.
buildpulls refs — kNN nearest assets to condition generation.
opsfeeds outcomes back — outcomes re-enter the space as the learning trace.

Contract: feeds all stages. The cortex is cross-cutting — it is not a stage in the pipeline, it is the space the pipeline lives in.

5Homeostasis role

This IS (X, d) from HOMEOSTASIS.md — the substrate where the Lyapunov error V = d(x, x*) is computed. Without it there is no measurable drift and therefore no self-healing: "on-brand" reverts to a vibe, the contraction Lᵢ < 1 cannot be checked, and reject/reroll has nothing to steer toward. Every other organ borrows its meaning of "correct" from here. Currently the space is duplicated across two Workers — taste-nim and the DESIGN_MEMORY_WORKER — so the setpoint and the index do not yet share one store. I3 unifies them; that single, shared aesthetic memory — readable at every scale — is the moat.
