Live Control Plane

  • cambium-quests owns quests, gate queue, bridge, handoffs.
  • curious.self renders the operator map and founder approvals.
  • Hermes EC2 polls and acks through scoped bridge credentials.

Memory Plane

  • cambium-cortex is the production semantic memory index.
  • Local cortex.db is the same contract as offline fallback.
  • Tenant and kind filters preserve memory boundaries.

Source Of Truth

  • Vault holds durable company docs and skill registry.
  • Paperclip supplies the inherited org/source contract.
  • Cambium records proof, quests, gates, and branch state.

External Boundaries

  • Composio supplies connected app tools, not policy.
  • Hosted providers sit behind the Cloudflare broker.
  • R2 thoughtseed-vault is encrypted backup, not live query.

Organ Spine

  • Genesis mints brand DNA and the x* setpoint.
  • Taste turns brand DNA into a gated taste brief.
  • Hands builds artifacts; Will turns artifacts into business.

Skill Clusters

  • skill-index.json is the registry of cluster capability.
  • resolve-task selects the cluster; conduct-execute dispatches work.
  • ship-battery, visual-qa, and cluster-health close the proof loop.

Route Inventory

  • The Worker exposes miniapp, ledger, gate, bridge, handoff, and provider broker routes.
  • Public reads are derived; writes require Telegram proof, bearer tokens, or HMAC envelopes.
  • The command panel shows status, agents, work, handoffs, and Hermes state.

Cloudflare State

  • QUESTS KV holds ledgers, gate queue, bridge messages, handoff tokens, and member indexes.
  • cambium-cortex is the shared Vectorize memory; sqlite remains the local contract twin.
  • Existing TeamForge D1/R2/Queue assets are audit or reuse candidates, not new authority.

AWS Boundary

  • EC2 is the always-on Hermes execution body, gateway, and runner timer.
  • Cloudflare keeps canonical state, gates, provider broker secrets, and public routes.
  • Direct provider keys stay out of the active EC2 runtime after broker cutover.

App Actions

  • Composio is the founder-connected app-action layer, not the policy authority.
  • Connected app surface includes GitHub, Gmail, Google Calendar, Outlook, Cal.com, Analytics, and Zoho Mail.
  • Agents should discover/execute through MCP/API or the CLI session, without starting OAuth flows.

Repo Ecosystem
