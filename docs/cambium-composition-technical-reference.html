1. Overview

Cambium is the composition layer that turns a constellation of specialized organs (genesis, taste, hands, will, cortex) into a single executable pipeline per tenant. It does not contain organ implementation — it plans, validates, invokes, and wires them using explicit, machine-readable contracts.

Core guarantee: every stage is a typed morphism. Output of stage N must satisfy the input contract of stage N+1, or the run fails closed before any irreversible spend.

2. The Pipeline (genesis → taste → build → ops + cortex)

The same self-similar shape recurs at every scale. The cross-cutting cortex (1024-dim NIM aesthetic memory) feeds every stage and receives outcomes for learning.

Agent note: brand_to_gtm and the C-suite dispatcher must preserve the original brand/copy/visual context. Flattening here breaks reference-anchored campaigns and the feedback loop into the cortex.

5. Spend Model & Fail-Closed Gates (Simplified Flow)

Only the explicit --approve path on a gated stage is allowed to spend. The hands stage resolution itself is free; dispatching real clusters is the gated extension.

6. For Agents, Conductors & Orchestrators (Critical)

Golden Rule: Forward the structured variable groups exactly as received. The handoff matrix (docs/plans/assets/stage-handoff-matrix.json) and prompt-reference-schema.json are the authoritative shapes for retrieval and execution.
Agent Consumption Checklist (every stage)
  • All required groups from the previous stage are present as top-level keys
  • No lossy summarization of brand_system, asset_plan, or acceptance_checks
  • Viability margin (solvency + mission-coherence) is positive before any new gated action
  • Any deviation (V increase) is recorded with the one-bit error-vs-intent signal and rationale
  • Output of this stage is a valid input for the declared downstream consumers

See also: the 10-task capability alignment plan (docs/plans/2026-06-09-...), INTEGRATION.md (I2/I3/I4), HOMEOSTASIS.md, and INFINITE-GAME.md.

7. Conductor Usage (bin/compose.mjs)

# Dry-run (never spends)
node bin/compose.mjs plan acme

# Execute only the free hands stage with a tasks file
node bin/compose.mjs run acme --stage build --execute --input examples/sample-tasks.md

# The only way to spend (explicit per-stage approval)
node bin/compose.mjs run acme --execute --approve taste

The conductor threads stage output → next stage input automatically. A refused gate breaks the chain and falls back to the declared input_default.
